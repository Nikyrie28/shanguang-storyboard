## 部署步骤

### 第一步：在 Supabase 创建项目

1. 访问 [supabase.com](https://supabase.com)
2. 点击 **New Project** 创建新项目
3. 设置项目名称（如：闪光分镜）
4. 选择区域（建议选择离你最近的区域）
5. 设置数据库密码（妥善保存）
6. 点击 **Create new project** 等待创建完成

### 第二步：创建数据库表

1. 进入新创建的项目
2. 左侧菜单选择 **SQL Editor**
3. 复制以下 SQL 并执行：

```sql
-- 1. 创建 profiles 表
CREATE TABLE IF NOT EXISTS public.profiles (
    id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
    email TEXT,
    username TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. 创建 projects 表
CREATE TABLE IF NOT EXISTS public.projects (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
    title TEXT NOT NULL,
    description TEXT DEFAULT '',
    password TEXT DEFAULT '',
    info JSONB DEFAULT '{"name":"","date":"","director":"","location":"","duration":""}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. 创建 project_rows 表
CREATE TABLE IF NOT EXISTS public.project_rows (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    project_id UUID REFERENCES public.projects(id) ON DELETE CASCADE NOT NULL,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
    row_order INTEGER DEFAULT 0,
    time TEXT DEFAULT '0:00 - 0:00',
    dept TEXT DEFAULT '',
    img TEXT DEFAULT '',
    content TEXT DEFAULT '',
    lens TEXT DEFAULT '',
    movement TEXT DEFAULT '',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. 创建索引
CREATE INDEX IF NOT EXISTS idx_projects_user_id ON public.projects(user_id);
CREATE INDEX IF NOT EXISTS idx_project_rows_project_id ON public.project_rows(project_id);
CREATE INDEX IF NOT EXISTS idx_project_rows_user_id ON public.project_rows(user_id);

-- 5. 启用 RLS
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.project_rows ENABLE ROW LEVEL SECURITY;

-- 6. RLS 策略
CREATE POLICY "Users can view own profile" ON public.profiles FOR SELECT USING (auth.uid() = id);
CREATE POLICY "Users can update own profile" ON public.profiles FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "Users can insert own profile" ON public.profiles FOR INSERT WITH CHECK (auth.uid() = id);

CREATE POLICY "Users can view own projects" ON public.projects FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can insert own projects" ON public.projects FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can update own projects" ON public.projects FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can delete own projects" ON public.projects FOR DELETE USING (auth.uid() = user_id);

CREATE POLICY "Users can view own rows" ON public.project_rows FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can insert own rows" ON public.project_rows FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can update own rows" ON public.project_rows FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can delete own rows" ON public.project_rows FOR DELETE USING (auth.uid() = user_id);

-- 7. 自动更新 updated_at
CREATE OR REPLACE FUNCTION update_updated_at_column() RETURNS TRIGGER AS $$ BEGIN NEW.updated_at = NOW(); RETURN NEW; END; $$ LANGUAGE plpgsql;

CREATE TRIGGER update_profiles_updated_at BEFORE UPDATE ON public.profiles FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
CREATE TRIGGER update_projects_updated_at BEFORE UPDATE ON public.projects FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
CREATE TRIGGER update_project_rows_updated_at BEFORE UPDATE ON public.project_rows FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

-- 8. 注册时自动创建 profile
CREATE OR REPLACE FUNCTION handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO public.profiles (id, email) VALUES (NEW.id, NEW.email); RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER;
CREATE OR REPLACE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```

### 第三步：获取 API 密钥

1. 在 Supabase 左侧菜单选择 **Settings** -> **API**
2. 复制 **Project URL** 和 **anon public** 密钥

### 第四步：配置前端连接信息

1. 复制 `config.example.js` 为 `config.js`
2. 填入你自己的 Supabase `Project URL` 和 `anon public` 密钥

```js
window.SHANGUANG_CONFIG = {
  supabaseUrl: 'https://your-project-id.supabase.co',
  supabaseAnonKey: 'your-anon-public-key'
};
```

3. `config.js` 已加入 `.gitignore`，避免把自己的项目配置误提交

### 第五步：本地测试

1. 使用任意静态服务器运行，例如：
```bash
# 方法1: 使用 npx serve
npx serve .

# 方法2: 使用 Python
python3 -m http.server 8000

# 方法3: 使用 VS Code Live Server 插件
# 右键 index.html -> Open with Live Server
```

2. 打开浏览器访问，测试注册和登录功能

### 第六步：部署到 Vercel（推荐，免费）

1. 注册 [Vercel](https://vercel.com) 账号（可用 GitHub 登录）
2. 点击 **New Project**
3. 选择 **Import Third-Party Git Repository** 或直接上传
4. 如果上传文件夹，需要创建 `vercel.json`：

```json
{
  "buildCommand": "",
  "outputDirectory": ".",
  "installCommand": "",
  "framework": null
}
```

5. 确保部署产物里包含正确的 `config.js`
6. 点击 Deploy

### 第七步：配置域名（可选）

Vercel 会自动分配一个免费域名（如：`your-project.vercel.app`）
你也可以绑定自己的域名。

当前版本使用 base64 存储参考图，简单但有以下限制：
- 单张图片太大会增加数据量
- 不适合大量图片

如需优化，可以后续接入 Supabase Storage。

## 常见问题

### Q: 注册后没收到验证邮件？
A: 检查垃圾箱，或在 Supabase Dashboard -> Authentication -> Settings 中关闭邮件确认。

### Q: 部署后无法登录？
A: 先检查 `config.js` 是否被正确上传，以及其中的 Supabase URL 和 anon key 是否填写正确。

### Q: 数据丢失怎么办？
A: 定期导出 JSON 备份，或接入 Supabase Storage 存储图片。
